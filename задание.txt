Основы программирования, практика - день 3
==========================================
Название проекта: objstore

Общая задача:
Построить приложение для хранения данных (структур), хранящее эти данные в виде XML-файла (базы данных на основе XML).
В качестве отправной точки смотри материал за уроки 9 и 13.

Схема работы:
Приложение выстраивается постепенно, необходимая функциональность будет указана ниже.

Присутствующие делятся на группы по 3-4 человека, 1 из которых назначается координатором работы. Каждый член группы берёт на себя часть задач для выполнения. После выполнения работы написанный код объединяется в единый проект, затем он доводится до работоспособного состояния.

Сами структуры берутся из задачника по номерам членов группы в списке. У каждой структуры помимо указанных в задаче полей должен быть числовой идентификатор (int ID)

Требуемая функциональность (в рекомендуемом порядке внедрения):
1). Возможность хранения данных в памяти для структуры одного типа (выбрать по номеру одного из членов группы)
2). Возможность просмотра содержимого базы данных через командную строку
3). Сохранение данных в виде XML-файла, их чтение из файла
4). Функция(процедура) для первичного наполнения базы тестовыми данными (3-4 записи)
5). Загрузка содержимого базы из файла на старте программы
6). Сохранение содержимого базы в конце работы программы
7). Возможность добавления записи в базу через командную строку (отдельно вводятся поля)
8). Возможность изменения записи в базе
9). Возможность удаления записи из базы (по идентификатору/индексу), С ЗАПРОСОМ НА ПОДТВЕРЖДЕНИЕ
10). Просмотр содержимого базы с сортировкой по указанному столбцу (в порядке убывания или возрастания - на выбор)
11). Очистка/обнуление базы, с подтверждением
12). Обработка ошибок: Проверка вводимых данных на корректность (в числовых полях - только числа, дата в определённом формате, и т.д.), сообщать пользователю об ошибке.
13). Возможность поиска по одному полю (выборка записей, у которых указанное поле содержит указанную строчку)
14). Экспорт данных в файл (с возможностью указания полного пути) с записями вида:
поле1, поле2, поле3, поле4, поле5;
15). Импорт данных из файла (с возможностью указания полного пути) с записями такого же вида
16). Хранение рабочих файлов: базы, журнала и конфигурации - в папке %APPDATA%\objstore\
17*). Экспорт содержимого базы в виде HTML-файла с таблицей
18*). Ведение журнала (лога) операций по изменению базы (и сохранения) в файле objstore.log
	Записи должны быть в стиле: 
	(чч:мм:сс) описание события
	При старте программы в лог пишется запись: (чч:мм:сс) Starting objstore. Current Date: ДД.ММ.ГГГГ
	При выходе из программы пишется похожая надпись о её остановке.
19*). Возможность хранения данных нескольких типов (структуры по номеру каждого члена группы) - сделать двухступенчатое меню - сначала выбирается тип структуры, затем операция. При выходе из меню операции возвращаемся назад в к выбору структуры.
20*). Конфигурация [из меню выбора структур]: рабочая директория по умолчанию (куда сохраняются экспортные файлы, если не указан полный путь)
21*). Конфигурация: значения полей по умолчанию (если пользователь при вводе оставляет поле пустым)
